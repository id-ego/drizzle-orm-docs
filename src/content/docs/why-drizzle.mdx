# Drizzle ORM
> Drizzle은 필요할 때 함께하고, 여유가 필요할 때는 물러서는 좋은 친구입니다.

Drizzle ORM은 머리가 있는 헤드리스 TypeScript ORM입니다. 🐲

보기에도 사용하기에도 간단하며, 프로젝트 _1000일차_에도 성능을 발휘하고,
여러분이 원하는 방식대로 작업할 수 있게 하며, 필요할 때 함께합니다.

**[관계형](/docs/rqb)과 [SQL-유사](/docs/select) 쿼리 API를 모두 제공하는 유일한 ORM**으로,
관계형 데이터에 접근할 때 두 세계의 장점을 모두 누릴 수 있습니다.
Drizzle은 가볍고, 고성능이며, 타입 안전하고, 유당 불포함, 글루텐 프리, 절제되어 있고, 유연하며 **설계부터 서버리스를 지원**합니다.
Drizzle은 단순한 라이브러리가 아닌, 하나의 경험입니다. 🤩

[![Drizzle bestofjs](@/assets/images/bestofjs.jpg)](https://bestofjs.org/projects/drizzle-orm)

## 헤드리스 ORM이란?
무엇보다도, Drizzle은 라이브러리이자 선택적으로 사용할 수 있는 보완 도구들의 모음입니다.

**ORM**은 _객체 관계 매핑(object relational mapping)_ 을 의미하며, 개발자들은 Django나 Spring과 같은 도구를 ORM이라고 부르는 경향이 있습니다.
우리는 이것이 레거시 명명법에 기반한 오해라고 진심으로 믿으며, 이러한 도구들을 **데이터 프레임워크**라고 부릅니다.

<Callout type="error" emoji="️💔">
  데이터 프레임워크를 사용하면 프로젝트를 **프레임워크 중심으로** 구축해야 하며, **프레임워크와 함께** 구축할 수 없습니다.
</Callout>

**Drizzle**은 프로젝트나 구조를 방해하지 않고 원하는 방식대로 프로젝트를 구축할 수 있게 합니다.

Drizzle을 사용하면 TypeScript로 데이터베이스 스키마를 정의하고 관리할 수 있으며, SQL-유사 방식이나
관계형 방식으로 데이터에 접근할 수 있고, 선택적 도구들을 활용하여
개발자 경험을 _최고 수준으로_ 끌어올릴 수 있습니다. 🤯 

## 왜 SQL-유사인가요?
**SQL을 알면, Drizzle을 아는 것입니다.**

다른 ORM과 데이터 프레임워크는 SQL에서 벗어나거나 추상화하려는 경향이 있어,
SQL과 프레임워크의 API를 모두 알아야 하는 이중 학습 곡선이 발생합니다.

Drizzle은 그 반대입니다.
우리는 SQL을 수용하고 Drizzle을 핵심부터 SQL-유사하게 만들어서,
학습 곡선이 거의 없거나 전혀 없이 SQL의 모든 기능에 접근할 수 있게 했습니다.

익숙한 **[SQL 스키마](/docs/sql-schema-declaration)**, **[쿼리](/docs/select)**,
**[자동 마이그레이션](/docs/migrations)** 그리고 **[한 가지 더](/docs/rqb)**를 제공합니다. ✨

<CodeTabs items={["index.ts", "schema.ts", "migration.sql"]}>
```typescript copy
// 데이터 접근
await db
	.select()
	.from(countries)
	.leftJoin(cities, eq(cities.countryId, countries.id))
	.where(eq(countries.id, 10))
```
```typescript copy
// 스키마 관리
export const countries = pgTable('countries', {
  id: serial('id').primaryKey(),
  name: varchar('name', { length: 256 }),
});

export const cities = pgTable('cities', {
  id: serial('id').primaryKey(),
  name: varchar('name', { length: 256 }),
  countryId: integer('country_id').references(() => countries.id),
});
```
```sql
-- 마이그레이션 생성
CREATE TABLE IF NOT EXISTS "countries" (
	"id" serial PRIMARY KEY NOT NULL,
	"name" varchar(256)
);

CREATE TABLE IF NOT EXISTS "cities" (
	"id" serial PRIMARY KEY NOT NULL,
	"name" varchar(256),
	"country_id" integer
);

ALTER TABLE "cities" ADD CONSTRAINT "cities_country_id_countries_id_fk" FOREIGN KEY ("country_id") REFERENCES "countries"("id") ON DELETE no action ON UPDATE no action;
```
</CodeTabs>

## 왜 SQL-유사가 아닌 것도 필요한가요?
우리는 항상 완벽하게 균형 잡힌 솔루션을 위해 노력하고 있으며, SQL-유사가 대부분의 요구사항을 충족하지만,
더 나은 방식으로 데이터를 쿼리할 수 있는 특정 일반적인 시나리오들이 있습니다.

우리는 **[Queries API](/docs/rqb)**를 구축했습니다. 이를 통해 가장 편리하고 성능이 좋은 방식으로 데이터베이스에서 관계형 중첩 데이터를 가져올 수 있으며,
조인과 데이터 매핑에 대해 고민할 필요가 없습니다.

**Drizzle은 항상 정확히 1개의 SQL 쿼리만 실행합니다.** 서버리스 데이터베이스와 함께 자유롭게 사용하고 성능이나 왕복 비용에 대해 걱정하지 마세요!

```ts
const result = await db.query.users.findMany({
	with: {
		posts: true
	},
});
```

## 서버리스?
<Callout type="info" emoji="🥳">
  가장 좋은 부분은 군더더기가 없다는 것입니다. **Drizzle은 정확히 0개의 의존성을 가지고 있습니다!**
</Callout>


![Drizzle is slim an Serverless ready](@/assets/images/drizzle31kb.jpg)

Drizzle ORM은 다이얼렉트별로 특화되어 있고, 가볍고, 고성능이며 **설계부터 서버리스를 지원**합니다.

우리는 Postgres, MySQL 등을 포함한 최고 수준의 SQL 다이얼렉트 지원을 제공하기 위해 많은 시간을 투자했습니다.

Drizzle은 업계 표준 데이터베이스 드라이버를 통해 기본적으로 작동합니다. 모든 주요 **[PostgreSQL](/docs/get-started-postgresql)**, **[MySQL](/docs/get-started-mysql)** 또는 **[SQLite](/docs/get-started-sqlite)** 드라이버를 지원하며, 새로운 드라이버를 **[정말 빠르게](https://twitter.com/DrizzleORM/status/1653082492742647811?s=20)** 추가하고 있습니다.
